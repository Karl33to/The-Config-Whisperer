Workflow

Paste or drop a config export into the Config Input box, then paste or drop a ruleset JSON into the Ruleset box and press Validate.

To build a ruleset from scratch: validate a config with no ruleset loaded, then click any line in the results to open the Rule Editor and assign rules. Use Export ruleset to save your work as a .json file that can be shared or re-imported later.

Ruleset Format

A ruleset is a JSON object with a rules key. Each entry maps a config line prefix to a rule: 

{
  "rules": {
    "set motor_pwm_rate": { "rule": "equals", "value": "480" },
    "aux 0": "allowed",
    "serial 0": { "rule": "review", "note": "Check port config" }
  }
}
Rule typeBehaviour
any value Line is permitted; value is not checked. Written as "allowed" in the ruleset JSON.
equals Remainder must exactly match the value field.
not equals Remainder must not equal the value field.
range Value must be numeric and fall within min and/or max bounds (either is optional).
forbidden Line must not appear in the config.
one of Value must exactly match one of the entries in the values array.
includes all Every entry in the values array must appear as a token in the value (order does not matter).
token set The value must contain exactly the tokens in the tokens array — no more, no less, any order.
positional Each entry in pos specifies an index and an allowed token list for that position in the remainder.
review Flags the line for manual inspection. An optional note is shown as a tooltip.

Add "required": true to any rule to fail validation when that line is absent from the config.
