Workflow

Paste or drop a Betaflight CLI export (diff all or preset save backup recommended) into the Config Input box, then paste or drop a ruleset JSON into the Ruleset box and press Validate.

To build a ruleset from scratch: validate a config with no ruleset loaded, then click any line in the results to open the Rule Editor and assign rules. Use Export ruleset to save your work as a .json file that can be shared or re-imported later.

Ruleset Format

A ruleset is a JSON object with a rules key. Each entry maps a config line prefix to a rule: 

{
  "rules": {
    "set motor_pwm_rate": { "rule": "equals", "value": "480" },
    "aux 0": { "rule": "allowed" },
    "serial 0": { "rule": "review", "note": "Check port config" }
  }
}
Rule typeBehaviour
allowed Line is permitted; value is not checked.
equals Remainder must exactly match the value field.
nonzero Value must not be 0.
forbidden Line must not appear in the config.
contains_all Every token in the contains_all array must appear somewhere in the line.
contains_any At least one token in the contains_any array must appear.
contains_exact The remainder must contain exactly the tokens in the tokens array — no more, no less.
positional Each entry in pos specifies an index and an allowed token list for that position in the remainder.
review Flags the line for manual inspection. An optional note is shown as a tooltip.

Add "required": true to any rule to fail validation when that line is absent from the config. A firmware key with an allowed_git_hashes array can be used to enforce a specific firmware build.
